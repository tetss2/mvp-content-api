# Architecture Overview

## High-level architecture

The system is evolving into a layered AI content operating platform.

Core architecture layers:

1. Retrieval layer
2. Knowledge ingestion layer
3. Runtime orchestration layer
4. Expert identity layer
5. Strategic generation layer
6. Telegram UX layer
7. Evaluation/reporting layer
8. AI workflow layer

---

# Retrieval layer

Responsible for:

- semantic retrieval
- vector search
- reranking
- deduplication
- retrieval evaluation
- context assembly

Primary components:

- FAISS indexes
- chunk metadata
- retrieval scripts
- retrieval reports

---

# Knowledge ingestion layer

Responsible for:

- source ingestion
- cleaning pipeline
- chunk generation
- metadata generation
- staging validation
- promote/rollback

Important principle:

Production retrieval should remain isolated from experimental ingestion.

---

# Runtime orchestration layer

Responsible for:

- generation orchestration
- strategic memory
- campaign continuity
- emotional pacing
- CTA pacing
- positioning
- authority management

Current evolving sublayers:

- campaign memory
- strategic brain
- editorial direction
- narrative loop tracking
- audience fatigue tracking

---

# Expert identity layer

Responsible for:

- author voice
- expert positioning
- style persistence
- identity continuity
- onboarding metadata

Important future direction:

Automated expert onboarding.

---

# Telegram UX layer

Responsible for:

- generation controls
- style selection
- content previews
- moderation feedback
- generation reports

---

# Evaluation/reporting layer

Responsible for:

- retrieval evaluation
- runtime verification
- preview reports
- orchestration reports
- campaign reports
- quality validation

# AI workflow layer

- architecture governance
