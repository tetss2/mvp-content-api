# Project Context

## Project

`mvp-content-api` is a Telegram-first AI content generation system for experts. The current product direction is an AI content operating system for expert-led media: Telegram bot UX, personalized expert knowledge bases, author voice capture, retrieval-augmented generation, runtime quality controls, and future Instagram/Telegram content automation.

## Current primary use case

The current expert/persona focus is Dinara Kachaeva: psychologist / sexologist. The system must generate expert content that uses source-grounded knowledge, author-like style, and controllable formats for Telegram/Instagram production.

## Core stack

- Node.js Telegram bot
- OpenAI API for generation and embeddings
- Local FAISS-based knowledge retrieval
- Knowledge ingestion and promotion scripts
- Author voice analysis and expert onboarding scripts
- Runtime generation adapter and safety/quality gates
- Railway-style production runtime target
- GitHub + Codex + ChatGPT workflow for development

## Product direction

The project is evolving from a simple RAG Telegram bot into a layered AI content orchestration engine. Important layers include:

- knowledge retrieval
- expert onboarding
- author identity runtime
- campaign memory
- strategic brain
- editorial direction
- runtime safety gates
- quality stabilization
- batch evaluation and reports

## Operating principle

ChatGPT acts as product/architecture planner and reviewer. Codex acts as implementation engine. GitHub is the source of truth for code, docs, iteration plans, and development state.
